## Diagram 2: Level 1 Data Flow Diagram

**What this shows:** The internal processes inside consec, showing how data moves between modules.

**Processes (rounded rectangles):**

| ID | Process | Description |
|----|---------|-------------|
| P1 | Scanner | Invokes Trivy, captures JSON output |
| P2 | Parser | Parses Trivy JSON and Dockerfile into structured objects |
| P3 | Ingestion Engine | Converts vulnerabilities to vectors, stores in ChromaDB |
| P4 | RAG Pipeline | Retrieves relevant CVEs, constructs prompt, queries LLM |
| P5 | Rules Engine | Runs 10 static Dockerfile security checks |
| P6 | Report Exporter | Generates Markdown and JSON reports |
| P7 | Display Engine | Formats output using Rich terminal tables and panels |

**Data Stores:**

| ID | Store |
|----|-------|
| D1 | ChromaDB Vector Store |
| D2 | Local Filesystem |

**External Entities:**

| ID | Entity |
|----|--------|
| E1 | Developer (User) |
| E2 | Trivy Scanner |
| E3 | Ollama LLM Server |

**Layer descriptions:**
- **CLI Layer:** User-facing commands (Typer framework). Entry point for all interactions.
- **Core Logic Layer:** Business logic вҖ” parsing, RAG, rules, export. Each module is independent.
- **Infrastructure Layer:** Connections to external systems вҖ” embeddings, LLM, config.
- **External Systems:** Trivy (subprocess), Ollama (HTTP), ChromaDB (file), Sentence-Transformers (library).

## Design Guidelines for All Diagrams

### Style

| Element | Style |
|---------|-------|
| Process (DFD) | Rounded rectangle, light fill |
| External entity | Rectangle, darker fill |
| Data store | Open-ended rectangle (parallel lines) |
| Decision (flowchart) | Diamond |
| Start/End | Rounded rectangle or oval |
| Action (flowchart) | Rectangle |
| Arrow | Solid line with arrowhead, labeled |
| System boundary | Dashed rectangle |

### Colors (optional for digital version)

| Element | Color |
|---------|-------|
| External systems | Light blue fill |
| Internal processes | Light green fill |
| Data stores | Light yellow fill |
| Decision diamonds | Light orange fill |
| Start/End | Gray fill |
| HIGH severity flow | Red accent |
| MEDIUM severity flow | Orange accent |
| LOW severity flow | Blue accent |
| Data flow arrows | Dark gray |

### Typography

- All labels: clean sans-serif (Arial, Helvetica, or Inter)
- Process names: bold, 12pt
- Data flow labels: regular, 10pt
- Notes/annotations: italic, 9pt

### Print Requirements

- Must be legible in black-and-white (use patterns/hatching instead of relying solely on color)
- Resolution: 300dpi minimum
- Page size: A4 (the diagrams should fit within A4 margins)
- Large diagrams (Level 1 DFD, Architecture): landscape orientation
- Smaller diagrams (individual flowcharts): portrait orientation
